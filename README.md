# Structured Urban Synthesis with Diffusion Transformers

This repository contains the implementation of a Diffusion Transformer (DiT) for synthesizing structured urban environments. The pipeline includes data preprocessing, Docker containerization, distributed training via Slurm, and hyperparameter optimization.

## Architecture and Usage

The system generates urban scenes by predicting noise across a sequence of denoising steps. The architecture replaces the standard U-Net backbone with a Transformer. It processes images as sequences of flattened patches. The model learns the spatial relationships of urban structures from these patches.

## Docker Environment

We use a Dockerfile to ensure consistent environments across local machines and the Quest compute cluster. The container is built upon the NVIDIA CUDA 11.8 base image. It installs PyTorch with GPU support and the required project dependencies.

To build and run the container locally:
```bash
docker build -t urban-dit .
docker run --gpus all -v $(pwd):/workspace -it urban-dit
```

## Preprocessing

The data pipeline processes raw urban driving datasets. The preprocessing script allocates the data into separate training, validation, and testing directories. It extracts and pairs the raw images with their corresponding structural masks.

Execution command:
```bash
python src/data/preprocess.py --data_dir data/ --output_dir processed_data/ --workers 8
```

## Training Infrastructure

Training executes on the Quest supercomputer using A100 GPU nodes. Slurm batch scripts handle module loading, environment activation, and GPU allocation. We use Weights & Biases (W&B) for metric logging and sweep management.

## Hyperparameter Sweep

We ran a 12-hour automated search using W&B sweep agents deployed across a Slurm job array. The agents optimized for validation loss (`val_loss`). 

Search parameters included:
* Learning rate (`lr`)
* Batch size (`batch_size`)
* Transformer depth (`depth`)
* Number of attention heads (`num_heads`)
* Patch size (`patch_size`)

### Experimental Results

The sweep generated specific findings regarding the model configuration:
* **Patch Size:** A smaller patch size of 16 produced lower validation loss compared to larger sizes.
* **Model Depth:** Increasing depth beyond 6 did not decrease loss within the evaluated limits.
* **Learning Rate:** The lowest loss configurations clustered at the extreme lower bound of the defined search space.

The optimal configuration identified by the sweep:
* `lr`: 0.000439
* `batch_size`: 64
* `depth`: 6
* `num_heads`: 12
* `patch_size`: 16

This setup reached a validation loss of 0.504 at epoch 24. We selected these parameters for the final 100-epoch training script.

## Assumptions and Future Issues

The experimental design relies on variables that require subsequent validation:

* **Metric Validity:** We use validation loss as a direct proxy for visual quality. The noise prediction error plateaued at 0.504. Diffusion models can continue developing high-frequency visual details after the loss curve flattens.
* **Local Minima:** The loss landscape for transformers contains saddle points. Terminating training based on early loss stagnation introduces the risk of missing steeper gradients later in the optimization process.
* **Parameter Independence:** The sweep optimized learning rate and batch size independently. Standard training practice dictates scaling these values proportionally.
* **Data Leakage:** If the preprocessing split allocated sequential frames from the same video sequence into both training and validation sets, the validation loss is artificially depressed. The model might be memorizing consecutive frames rather than learning to generalize urban structures.
